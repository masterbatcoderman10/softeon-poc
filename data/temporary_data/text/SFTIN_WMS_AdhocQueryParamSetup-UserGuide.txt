Adhoc Query Param Setup 
1. Overview 
You can use the Adhoc Query Param Setup screen for defining the parameters used in an Adhoc query 
and its attributes such as the sequence in which it is used in the Query, the Object that represents the 
parameter in the UI, whether the parameter is mandatory or not. 
Menu Access: ADM – Utility > Adhoc Setup > Adhoc Query Param Setup 
2. Pre-requisites 
• 
Object ID must be created and mapped with Definition ID 
3. Defining Parameters 
Mandatory Values 
1) Specify the Query ID to which the parameter belongs. 
2) Specify/select the UI Definition ID. 
3) Specify/select the Object ID. 
4) Specify the Seq No (Sequence Number), the numerical order of the input parameter available in 
the query.  
5) Specify the expression in SQL Placeholder Expr. 
6) Specify the Condition/ value in SQL Placeholder Value when the expression is executed in the 
SQL. 
The system will display the result of the condition (SQL Placeholder Value) in the grid when the 
expression (SQL Placeholder Expr) is executed in the SQL. 
 
Figure 1 – Adhoc Query Param Setup screen 
Optional Values 
7) Specify the Param Caption, the caption for the control to be displayed in the Adhoc Query GUI. 
8) Specify the Default Value to be displayed in the control(s) on form load of the screen. 
9) Select the Mandatory Flag check box, to indicate the Query contains SQL operator. The Query 
that has only LIKE clause need not be mandatory.
